# AlphaMining 日频复现运行手册

## 1. 适用范围

本手册用于从零运行日频版 GFlowNet + AlphaEval + LightGBM + RQAlphaPlus 流水线。行情数据只从本地 `data/price.csv` 读取，不调用外部数据接口。

完整流程如下：

```text
price.csv
  ↓
数据检查、复权、异常处理与截面标准化
  ↓
表达式引擎与 GFlowNet 因子挖掘
  ↓
AlphaEval 评价与 DPP 去相关筛选
  ↓
LightGBM 滚动融合，预测 t+1 到 t+5 收益
  ↓
RQAlphaPlus Top N 等权策略回测
```

## 2. 运行环境分工

| 阶段 | 推荐环境 | 必要条件 |
|---|---|---|
| 数据准备、表达式检查、单元测试 | 本地电脑或 Colab | Python 3.10 及以上 |
| GFlowNet 正式训练 | Google Colab | NVIDIA A100、CUDA、PyTorch |
| AlphaEval 与 LightGBM | Colab 或内存充足的本地环境 | 已生成因子池和因子矩阵 |
| 策略回测 | 已获授权的 RQAlphaPlus 环境 | `rqalpha_plus` 与本地数据包 |

RQAlphaPlus 不在 `requirements.txt` 中，因为它是授权软件，必须通过米筐授权渠道单独安装。

## 3. 获取代码

在本地终端或 Colab 中执行：

```bash
git clone https://github.com/JacksonChiy/AlphaMining-GFlowNet-AlphaEval.git
cd AlphaMining-GFlowNet-AlphaEval
```

Notebook 默认使用上述地址。如果需要运行其他 fork 或分支，可在执行 clone 单元格前设置：


## 5. 准备 `price.csv`

### 5.1 文件位置

将私有行情文件复制或上传到：

```text
data/price.csv
```

该文件已被 `.gitignore` 忽略，不会进入 Git 提交。

### 5.2 字段要求

| 规范字段 | 必需 | 常见可识别别名示例 | 说明 |
|---|---:|---|---|
| `date` | 是 | `trade_date`、`datetime`、`日期`、`交易日期` | 交易日期 |
| `code` | 是 | `symbol`、`ticker`、`证券代码`、`股票代码` | 股票标识 |
| `open` | 是 | `open_price`、`开盘价` | 开盘价 |
| `high` | 是 | `high_price`、`最高价` | 最高价 |
| `low` | 是 | `low_price`、`最低价` | 最低价 |
| `close` | 是 | `close_price`、`收盘价` | 收盘价 |
| `volume` | 是 | `vol`、`trade_volume`、`成交量` | 成交量 |
| `amount` | 否 | `turnover`、`trade_amount`、`成交额` | 成交额 |
| `vwap` | 否 | `avg_price`、`average_price`、`成交均价` | VWAP；缺失时自动估算 |
| `adj_factor` | 否 | `adjust_factor`、`factor`、`复权因子` | 时点可得的复权因子 |
| `industry` | 否 | `industry_code`、`申万行业`、`行业` | 行业风险暴露 |
| `market_cap` | 否 | `total_market_cap`、`float_market_cap`、`市值` | 市值风险暴露 |

最小示例：

```csv
date,code,open,high,low,close,volume,amount
2020-01-02,000001.SZ,16.65,16.95,16.55,16.87,153023500,2578400000
2020-01-03,000001.SZ,16.94,17.31,16.92,17.18,111619400,1911550000
```

### 5.3 数据边界

- 每一行应代表一只股票的一个交易日；
- 同一 `date + code` 若重复，保留最后一行；
- 价格必须为正，且 `low <= high`；
- 缺失价格只在同一股票内部前向填充；
- `adj_factor` 必须是当时已经可得的数据，不能使用事后修订信息；
- 行业和市值字段缺失时可以运行，但对应风险惩罚不会启用；
- LightGBM 默认至少需要约 504 个训练交易日，建议准备明显更长的历史区间。

## 6. 阶段一：数据预处理

推荐在 Colab 中打开并运行：

```text
notebooks/01_data_prepare.ipynb
```

也可在项目根目录单独执行：

```bash
python -c "from src.data_loader import prepare_price_csv; prepare_price_csv()"
```

处理内容包括：日期转换、字段自动映射、稳定排序、去重、缺失值与异常值处理、可选复权、同日缩尾和截面 z-score。

成功后检查：

```text
data/daily_price.pkl
results/data_quality_report.json
```

重点查看质量报告中的输入/输出行数、字段映射、删除的无效行、异常值截断数量、是否执行复权，以及行业和市值字段是否可用。

## 7. 阶段二：表达式引擎检查

运行：

```text
notebooks/02_expression_engine.ipynb
```

该 Notebook 会随机生成表达式，完成序列化与反序列化，并在 `daily_price.pkl` 上执行因子计算。典型表达式为：

```text
cs_rank(ts_mean(close,20))
```

若此阶段出现大量空值，先确认所选时间窗口之前存在足够历史数据；窗口初期出现空值属于正常现象。

## 8. 阶段三：在 Colab A100 训练 GFlowNet

### 8.1 选择 A100

在 Google Colab 菜单中选择“运行时 → 更改运行时类型”，将硬件加速器设置为 **A100 GPU**。若账号当前无法分配 A100，请稍后重试或使用支持 A100 的 Colab 方案；正式实验不要跳过硬件校验。

### 8.2 上传数据

clone 仓库后，把本地 `price.csv` 上传到 Colab 会话中的：

```text
AlphaMining-GFlowNet-AlphaEval/data/price.csv
```

Colab 临时磁盘会随会话释放，重要结果应及时下载或复制到已挂载的持久存储。

### 8.3 运行训练 Notebook

打开：

```text
notebooks/03_train_gflownet_A100.ipynb
```

按顺序运行全部单元格。GPU 检查单元格必须输出：

- `CUDA available: True`；
- GPU 型号包含 `A100`；
- GPU 总显存；
- CUDA 与 PyTorch 版本。

默认训练参数来自 `configs/training_config.yaml`：

| 参数 | 默认值 |
|---|---:|
| 隐藏维度 | 256 |
| Transformer 层数 | 4 |
| 注意力头数 | 8 |
| 训练轮数 | 100 |
| 每轮轨迹数 | 32 |
| 学习率 | 0.0001 |
| 混合精度 | 开启 |
| 随机种子 | 42 |

正式训练会保存并重新加载检查点。成功标志：

```text
checkpoints/gflownet_best.pt
results/gflownet_training_metrics.csv
results/alpha_pool.csv
results/alpha_factor_matrix.pkl
```

`alpha_pool.csv` 记录 `factor_001`、`factor_002` 等因子名称、表达式和奖励信息；`alpha_factor_matrix.pkl` 保存按日期和股票索引的因子值。

## 9. 阶段四：AlphaEval 筛选

运行：

```text
notebooks/04_alpha_eval.ipynb
```

评价包括预测能力、滚动稳定性、扰动鲁棒性、表达式逻辑性、RRE 和 DPP 多样性筛选。未来收益标签严格定义为：

```text
close(t+5) / close(t+1) - 1
```

该定义表示使用 `t` 日收盘前可形成的因子，预测从下一交易日到第 5 个未来交易日的收益区间，避免把 `t` 日收盘价直接当作可成交起点。

成功后检查：

```text
results/alpha_eval_result.csv
```

重点字段包括 `factor`、`IC`、`RankIC`、`ICIR`、`Sharpe`、`complexity`、`score` 和 `dpp_selected`。后续 LightGBM 默认只使用 DPP 选中的因子。

## 10. 阶段五：LightGBM 因子融合

运行：

```text
notebooks/05_lgbm_model.ipynb
```

默认配置使用 756 个交易日滚动训练窗、至少 504 个训练日、每 20 个交易日重训一次，并在训练与预测之间保留 5 个交易日 purge 间隔。

成功后检查：

```text
results/lightgbm/lgbm_model.joblib
results/lightgbm/model_metrics.csv
results/lightgbm/feature_importance.csv
results/lightgbm/prediction_score.csv
```

`prediction_score.csv` 是回测输入，包含信号日期、股票代码、预测分数和当日截面排名。

## 11. 一条命令运行前五阶段

在 A100 环境、数据和依赖均准备完毕后，可执行：

```bash
python -m scripts.run_daily_pipeline --pool-size 100
```

正式运行默认强制检测 A100。仅做很小规模的本地代码路径检查时可以使用：

```bash
python -m scripts.run_daily_pipeline --pool-size 5 --allow-non-a100
```

该选项不适用于正式训练或研究结果发布。

## 12. 阶段六：RQAlphaPlus 回测

### 12.1 前提

在已获授权的环境中安装 `rqalpha_plus`，准备好本地数据包，并确认 `results/lightgbm/prediction_score.csv` 已生成。本项目不会自动下载 RQAlphaPlus 数据包。

### 12.2 执行

```bash
python -m rqalpha_strategy.run_backtest \
  --bundle ~/.rqalpha-plus/bundle \
  --predictions results/lightgbm/prediction_score.csv \
  --output-dir results/backtest_report \
  --initial-cash 1000000 \
  --benchmark 000300.XSHG \
  --top-n 30 \
  --rebalance-days 5 \
  --slippage 0.001
```

也可在前五阶段的一键命令中附加：

```bash
python -m scripts.run_daily_pipeline \
  --pool-size 100 \
  --rqalpha-bundle ~/.rqalpha-plus/bundle
```

### 12.3 防止未来数据泄漏

策略只选择 `signal_date < trade_date` 的最近信号。同日收盘数据生成的分数不会在同一根日线成交。RQAlphaPlus 负责 T+1、整手约束、涨跌停、成交量限制、佣金、印花税、滑点及成交记录。

### 12.4 回测输出

成功后，`results/backtest_report/` 至少应包含：

```text
backtest_result.pkl
backtest_summary.json
equity_curve.png
```

RQAlphaPlus 分析器还会生成可用的绩效报告、持仓和交易明细。重点核对年度收益、总收益、Sharpe、最大回撤、波动率、换手率、净值曲线与交易历史。

## 15. 常见问题

### 15.1 提示没有找到 `data/price.csv`

确认当前目录是仓库根目录，并确认文件路径严格为 `data/price.csv`。Colab 每次重新连接后可能需要重新上传数据。

### 15.2 提示缺少必需字段

打开 `price.csv` 第一行，检查日期、代码、开高低收和成交量字段是否存在。若字段名不在自动别名表中，可先改为规范名称；不要把索引列误当作股票代码。

### 15.3 CUDA 不可用或 GPU 不是 A100

在 Colab 重新选择 A100 运行时，然后从 GPU 检查单元格重新执行。若无法分配 A100，请不要用 `--allow-non-a100` 生成正式结果。

### 15.4 macOS 导入 LightGBM 失败

先执行 `brew install libomp`，重启终端并重新激活虚拟环境。仍失败时重新安装 LightGBM wheel。

### 15.5 AlphaEval 或 LightGBM 可用样本过少

确认股票数量足够、历史长度明显超过 504 个交易日，并检查大窗口因子是否产生过多空值。行业、市值字段缺失不会阻止运行。

### 15.6 RQAlphaPlus 无法导入

这不是公开 `pip` 依赖缺失。请切换到已通过米筐授权渠道安装 `rqalpha_plus` 的 Python 环境。

### 15.7 回测没有交易

检查预测文件是否包含 `signal_date`、`code`、`prediction_score`，日期是否覆盖 RQAlphaPlus 数据包，股票代码能否映射为 `.XSHG`、`.XSHE` 或 `.XBSE`，以及第一天没有更早信号是否属于预期。
